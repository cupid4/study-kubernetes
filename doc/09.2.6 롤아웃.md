# 9.2.6 롤아웃 (Rollout) - 컨테이너 정보 및 버전 확인

쿠버네티스에서 롤아웃(Rollout)은 쉽게 말해 "실행 중인 컨테이너의 이미지 버전(태그)을 바꾸는 과정 전체"를 의미합니다.
(이미 배포되어 있는 컨테이너의 버전을 업데이트한다.)
조금 더 구체적으로는 이미지 버전이 바뀔 때 쿠버네티스가 제공하는 다음과 같은 강력한 내장 기능들을 통틀어 롤아웃이라고 부릅니다.

---

### 1. 롤아웃이 해주는 핵심 작업 3가지

* **무중단 교체 (Rolling Update):** 지금 교재 289페이지에서 보신 것처럼, 기존 버전(v1.24) 파드를 한 번에 다 죽여서 서비스를 끊지 않고, 새 버전(v1.25) 파드를 하나씩 띄우면서 구버전을 차례대로 정리해 줍니다. 사용자는 배포 중인지도 모르게 서비스가 유지됩니다.
* **이력 관리 (Revision History):** 버전이 바뀔 때마다 `revision 1`, `revision 2`처럼 클러스터에 기록을 남깁니다. (아까 `describe`에서 확인하신 부분입니다.)
* **되돌리기 (Rollback):** 만약 새 버전 이미지로 바꿨는데 예상치 못한 버그가 발생했다면, 명령어 한 줄(`kubectl rollout undo`)로 이전 안정적인 버전으로 순식간에 복구할 수 있습니다.

---

### 💡 딱 한 줄 요약

**컨테이너 이미지 버전을 업데이트하고, 배포 과정을 관리하며, 필요할 때 전 버전으로 되돌리는 일련의 안전 장치 매커니즘** 이 바로 롤아웃입니다.


## 1. 매니페스트 배포 및 파드 확인

새로운 설정 파일인 `deploy-test03.yml`을 클러스터에 적용하고 파드가 정상 구동되었는지 확인합니다.

* **실행 명령어:**
```bash
kubectl apply -f deploy-test03.yml ⑪
# 출력: deployment.apps/deploy-test01 created

kubectl get pod ⑫

```


* **결과:** `deploy-test01-6d85bdf7fd-...` 형태의 이름을 가진 3개의 파드가 정상적으로 생성되어 `Running` 상태로 작동하는 것을 볼 수 있습니다.

---

## 2. 디플로이먼트 상세 조회를 통한 Nginx 버전 확인 (`describe`)

배포된 파드들이 정확히 어떤 버전의 이미지를 사용하고 있는지 확인하기 위해 디플로이먼트의 상세 명세를 조회합니다.

* **실행 명령어:**
```bash
kubectl describe deployment deploy-test01 ⑪

```


* **핵심 출력 데이터 분석:**
```text
Name:                   deploy-test01
Selector:               app.kubernetes.io/name=web-deploy
Replicas:               3 desired | 3 updated | 3 total | 3 available | 0 unavailable
StrategyType:           RollingUpdate
RollingUpdateStrategy:  25% max unavailable, 25% max surge
Pod Template:
  Containers:
    nginx:
      Image:            nginx:1.24 ⑫

```


* **StrategyType:** 기본 배포 전략으로 **`RollingUpdate`**(무중단 점진적 업데이트)가 설정되어 있음을 보여줍니다.
* **Image (⑫):** 현재 파드 안에서 실행 중인 Nginx 웹서버의 버전이 `1.24`인 것을 명확하게 확인할 수 있습니다.
