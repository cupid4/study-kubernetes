# 9.2.3 레플리카셋 조정

## 1. 레플리카셋 조정을 위한 기본 원칙

* 레플리카셋(ReplicaSet)은 사용자가 지정한 파드(Pod)의 개수를 그대로 유지하고 가용성을 보장하는 컨트롤러입니다.
* 쿠버네티스 운영 시 파드의 개수를 늘리거나 줄여야 할 때, **레플리카셋 오브젝트를 직접 수정하거나 다루지 않는 것이 원칙**입니다.
* 상위 관리자인 **디플로이먼트(Deployment)를 통해** 파드의 개수를 조절하면, 디플로이먼트가 백엔드에서 레플리카셋을 알아서 안전하게 조정합니다.

---

## 2. 명령어로 레플리카셋 개수 지정하여 배포하기

디플로이먼트를 처음 생성할 때 `--replicas` 옵션을 사용하면 원하는 파드의 개수를 명시하여 배포할 수 있습니다.

* **실행 명령어:**
```bash
kubectl create deployment deploy-nginx --image=nginx --replicas=3

```


* `--replicas=3`: 이 옵션을 통해 nginx 웹서버 파드를 처음부터 동시에 3개 구동하도록 지정합니다.
* 실행 결과로 `deployment.apps/deploy-nginx created` 메시지가 출력되며 디플로이먼트가 정상 생성됩니다.



---

### 💡 핵심 요약

1. 파드의 수량을 관리하는 것은 **레플리카셋**의 역할이다.
2. 하지만 실무에서는 레플리카셋을 직접 건드리지 않고, 항상 **디플로이먼트를 통해서 간접적으로 수량을 조절**한다.
3. 처음 디플로이먼트를 만들 때 **`--replicas=개수`** 옵션을 주면 원하는 수량만큼 파드를 동시에 띄울 수 있다.
