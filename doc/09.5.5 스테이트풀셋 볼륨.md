# 9.5.5 스테이트풀셋 볼륨

![alt text](<asset/09.5.5 스테이트풀셋 볼륨/image.png>)


그림은 **헤드리스 서비스(왼쪽)**, **볼륨 기능이 추가된 스테이트풀셋(가운데)**, 그리고 **실제 저장소인 PersistentVolume(오른쪽)** 이 서로 어떤 이름과 속성을 매칭하여 하나로 움직이는지 보여주는 전체 구조도입니다.

#### 3줄 요약

1. **서비스(왼쪽) $\leftrightarrow$ 스테이트풀셋(가운데):** `web-sfs01`(라벨)과 `sfs-service01`(이름)로 네트워크 통로를 연결한다.
2. **컨테이너 내부 $\leftrightarrow$ 볼륨틀(가운데 안):** `sfs-vol01`이라는 이름으로 파드 안의 폴더와 저장소 요청을 묶어준다.
3. **볼륨틀(가운데) $\leftrightarrow$ 실제저장소(오른쪽):** `pv-sfs-test01`이라는 `storageClassName`을 매칭하여 실제 하드웨어 디스크를 확보한다.



---

## 1. 네트워크 연결 (보라색 & 초록색 상자)

가운데 스테이트풀셋이 외부와 통신하고 고정된 주소를 갖기 위해 왼쪽의 서비스와 연결되는 과정입니다. (이전 단계에서 보았던 구조와 같습니다.)

* **보라색 상자 (`serviceName`):** 스테이트풀셋 내부에서 `serviceName: sfs-service01`을 지정하여 왼쪽 서비스의 이름(`name: sfs-service01`)을 조준합니다. 이로 인해 파드들에게 고정된 도메인 주소가 부여됩니다.
* **초록색 상자 (`Labels`):** 왼쪽 서비스는 `web-sfs01` 명찰을 가진 파드를 찾고(`selector`), 스테이트풀셋은 붕어빵 틀(`template`)을 통해 태어날 파드에게 똑같이 `web-sfs01` 명찰을 붙여주어 서로 한 팀이 됩니다.

---

## 2. 스테이트풀셋 내부의 볼륨 연결 (살구색 상자)

가운데 파일 안에서 "컨테이너 내부 폴더"와 "자동 생성될 저장소 요청서(PVC)"를 한 몸으로 묶어주는 내부 연결입니다.

* **`volumeMounts`의 name (`sfs-vol01`):** 엔지니어가 컨테이너 내부의 `/mount01`이라는 디렉토리를 외부 저장소와 연결하고 싶어 합니다. 이때 "이 연결 통로의 이름을 `sfs-vol01`이라고 부를게" 하고 지정한 것입니다.
* **`volumeClaimTemplates`의 name (`sfs-vol01`):** 스테이트풀셋이 파드를 만들 때 자동으로 저장소 요청서(PVC)를 찍어내는 틀입니다. 이 틀의 이름을 위의 마운트 이름과 똑같이 `sfs-vol01`로 맞춰놓음으로써, 컨테이너 내부의 `/mount01` 폴더가 이 틀을 통해 만들어질 저장소와 완벽하게 이어지게 됩니다.

---

## 3. 실제 저장소 하드웨어와의 연결 (주황색 상자)

이제 자동으로 만들어진 저장소 요청서(가운데)가 실제 서버에 있는 물리적인 저장 공간(오른쪽)을 찾아가서 결합하는 가장 중요한 연결입니다.

* **가운데 (`storageClassName: pv-sfs-test01`):** 스테이트풀셋의 저장소 틀에서 "나는 `pv-sfs-test01`이라는 종류(클래스)의 저장소가 필요해!"라고 요청합니다.
* **오른쪽 (`storageClassName: pv-sfs-test01`):** 마침 오른쪽에 미리 만들어 둔 실제 저장소인 PersistentVolume(PV)에 "내 종류(클래스)는 `pv-sfs-test01`이야"라고 똑같은 이름이 적혀 있습니다.
* **매칭 결과:** 쿠버네티스는 이 `storageClassName`의 이름이 일치하는 것을 확인하고, 가운데 틀을 통해 만들어진 요청서(PVC)를 오른쪽의 실제 저장소(PV)와 끈끈하게 연결(`Binding`)해 줍니다.

> 💡 **결과적으로 무슨 일이 일어날까요?**
> 오른쪽 PV에 적힌 실제 서버의 경로(`hostPath.path: /home/eevee/...`)가 가운데 스테이트풀셋 컨테이너 안의 `/mount01` 폴더와 일직선으로 연결됩니다. 덕분에 컨테이너 안에서 파일을 저장하면 실제 서버의 저 경로에 안전하게 데이터가 저장됩니다.
