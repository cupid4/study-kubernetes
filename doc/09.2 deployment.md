# 9.2 디플로이먼트 (Deployment)

## 1. 디플로이먼트의 개념

쿠버네티스에서 **파드(Pod)를 효율적으로 관리하고 배포**하기 위해 사용하는 상위 수준의 컨트롤러(Controller)입니다.
**파드 개수를 조절** 하려면 레플리카셋을 직접 다루는 것이 아니라 **디플로이먼트를 통해 수행**한다.

* **레플리카셋(ReplicaSet)과의 관계:**
* 디플로이먼트를 이해하려면 먼저 **레플리카셋**을 알아야 합니다.
* 레플리카셋은 사용자가 정의한 **파드의 개수를 항상 동일하게 유지하고 가용성을 보장**하는 역할을 합니다.
  * 원하는 파드 개수만큼 유지시켜주는 역할을 하는 컨트롤러
* 개발자가 레플리카셋을 직접 하나하나 만지는 대신, **디플로이먼트가 백엔드에서 레플리카셋을 자동으로 관리**해 줍니다.
* 즉, 관계의 계층은 다음과 같습니다:

$$\text{디플로이먼트 (Deployment)} \rightarrow \text{레플리카셋 (ReplicaSet)} \rightarrow \text{파드 (Pod)}$$

## 2. 디플로이먼트 생성 및 실행 (`kubectl create deployment`)

명령어를 사용하여 YAML 파일 없이 CLI 환경에서 즉시 디플로이먼트를 생성할 수 있습니다.

* **실행 명령어:**
```bash
kubectl create deployment deploy-hello --image=hello-world
# 출력: deployment.apps/deploy-hello created

```


* **클러스터 상태 확인 (`kubectl get all`):**
디플로이먼트를 생성한 후 클러스터 내부의 전체 오브젝트 상태를 조회하면 다음과 같이 나타납니다.
```bash
kubectl get all
NAME                                READY   STATUS              RESTARTS   AGE
pod/deploy-hello-54df7ff57c-qgncl   0/1     CrashLoopBackOff    4 (89s ago) 3m4s

```



### ⚠️ 결과 분석 (왜 CrashLoopBackOff가 뜨나요?)

* 앞서 실습했던 단독 파드 생성 때와 마찬가지로, `hello-world` 이미지는 일회성 메시지만 출력하고 바로 종료되는 구조입니다.
* 디플로이먼트는 자신이 관리하는 파드가 종료되면 **"어? 파드가 죽었네? 지정된 개수를 유지해야 하니까 다시 살려야지!"** 하고 계속해서 재시작을 시도합니다.
* 이 재시작 시도가 반복되면서 상태가 `CrashLoopBackOff`로 표시되는 것이며, 이는 디플로이먼트의 파드 유지 메커니즘이 정상적으로 작동하고 있음을 보여주는 현상입니다.

### 1. 다중 리소스 동시 조회 (`kubectl get deploy,rs,po`)

쿠버네티스에서는 여러 종류의 리소스를 쉼표(`,`)로 연결하여 한 번에 조회할 수 있습니다.

* **핵심 문법 규칙:** 쉼표 뒤에 **공백(띄어쓰기)이 없어야** 합니다.
* `deploy,rs,po` (O)
* `deploy, rs, po` (X $\rightarrow$ 구문 에러 발생)


* **출력 결과:** 디플로이먼트(Deployment), 레플리카셋(ReplicaSet), 파드(Pod)의 상태와 관계를 한눈에 파악할 수 있어 클러스터 상태 점검 시 매우 자주 사용됩니다.

### 2. 쿠버네티스 리소스 주요 약어(Shortnames)

명령어를 빠르게 입력할 수 있도록 쿠버네티스에서 공식적으로 제공하는 줄임말입니다.

| 전체 리소스명 | 약어 (Shortname) |
| --- | --- |
| **`deployment`** | `deploy` |
| **`replicaset`** | `rs` |
| **`pod`** | `po` |

### 3. 디플로이먼트 삭제 (`kubectl delete`)

디플로이먼트를 삭제하면 해당 디플로이먼트가 하위에 자동으로 생성했던 레플리카셋과 파드까지 연쇄적으로 깔끔하게 삭제됩니다.

* **삭제 명령어:**
```bash
kubectl delete deployment deploy-hello

```


* **삭제 확인:** 삭제 후 `kubectl get deploy,rs,po`를 실행하면 `No resources found in default namespace.` 문구가 출력되며 모든 관련 자원이 정상적으로 정리된 것을 확인할 수 있습니다.
