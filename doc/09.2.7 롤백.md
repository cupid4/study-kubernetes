# 9.2.7 롤백 (Rollback)

쿠버네티스에서 롤백(Rollback)이란 롤아웃(업데이트)을 수행한 후, 예상치 못한 버그나 장애가 발생했을 때 **이전의 안정적인 컨테이너 상태로 되돌리는 것**을 의미합니다.

## 1. 롤백 명령 실행 (`kubectl rollout undo`)

디플로이먼트의 설정을 업데이트하기 전 버전으로 되돌릴 때 `kubectl rollout undo` 명령어를 사용합니다.

* **실행 명령어:**
```bash
kubectl rollout undo deployment deploy-test01
# 출력: deployment.apps/deploy-test01 rolled back

```


* **결과:** 명령어 한 줄로 즉시 롤백 프로세스가 시작되며, 백엔드에서는 신형 파드를 줄이고 구형 파드를 다시 띄우는 역방향 롤아웃이 진행됩니다.

---

## 2. 롤백 후 파드 상태 확인 (`kubectl get pod`)

명령어를 내린 직후 파드의 상태를 확인하면, 이전 버전의 레플리카셋 해시값(`6d85bdf7fd`)을 가진 파드들이 새롭게 생성되어 돌아가고 있는 것을 볼 수 있습니다.

```bash
eevee@myserver01:~/work/ch09/ex04$ kubectl get pod ❷
NAME                            READY   STATUS    RESTARTS   AGE
deploy-test01-6d85bdf7fd-dj249   1/1     Running   0          7s
deploy-test01-6d85bdf7fd-dxg4g   1/1     Running   0          5s
deploy-test01-6d85bdf7fd-mqgcs   1/1     Running   0          4s

```

* **해시값의 비밀:** 파드 이름 중간에 있는 `6d85bdf7fd`는 Nginx `1.24` 버전을 배포했던 첫 번째 레플리카셋의 고유 ID입니다. 즉, `1.25` 버전 파드들이 사라지고 예전 파드들이 완벽하게 복구되었음을 의미합니다.

---

## 3. 디플로이먼트 상세 정보 확인 (`describe`)

상세 조회를 통해 롤백이 최종적으로 어떻게 기록되었는지 검증합니다.

```bash
eevee@myserver01:~/work/ch09/ex04$ kubectl describe deployment deploy-test01 ❸

```

* **Annotations:** `deployment.kubernetes.io/revision: 3`으로 변경되어 있습니다. 업데이트와 롤백 과정을 거치며 이 디플로이먼트의 세 번째 수정 이력(Revision 3)이 생성된 것입니다.
* **Pod Template (배포 사양):** 컨테이너 이미지 정보가 최신 버전(`1.25`)에서 다시 이전 버전인 `nginx:1.24`로 안전하게 되돌아온 것을 확인할 수 있습니다.

---

### 💡 핵심 요약

1. **`kubectl rollout undo`** 명령어를 사용하면 복잡한 수정 과정 없이 단 한 줄로 이전 버전으로 시스템을 되돌릴 수 있습니다.
2. 롤백 역시 서비스 중단 없이 **무중단(Rolling Update) 방식으로 안전하게 진행**됩니다.
3. 롤백이 완료되면 디플로이먼트의 `revision` 번호는 하나 증가하며, 파드들은 구버전의 레플리카셋 하위로 재배치됩니다.
