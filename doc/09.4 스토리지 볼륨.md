## 9.4.1 쿠버네티스 스토리지 볼륨의 3가지 핵심 유형
![alt text](<asset/09.4 스토리지 볼륨/image.png>)


### 1. 📂 emptyDir (임시 볼륨)

* **개념:** 파드 내부에서 **일시적(임시적)으로 사용하는 볼륨**입니다.
* **특징:** * 파드가 노드에 처음 할당될 때 생성되며, 초기 상태는 이름 그대로 비어 있는 상태(Empty)로 시작합니다.
* 파드 내에 여러 개의 컨테이너가 존재할 경우, 이 컨테이너들이 **동일한 파일을 서로 읽고 쓸 수 있는 공유 공간** 역할을 합니다.


* **수명:** **파드의 수명과 운명을 같이 합니다.** 파드가 노드에서 삭제되면 `emptyDir` 내부에 저장되어 있던 데이터도 함께 깨끗이 삭제됩니다. (※ 컨테이너가 재시작하는 것으로는 삭제되지 않지만, 파드 자체가 날아가면 데이터도 날아갑니다.)

### 2. 💻 hostPath (노드 로컬 볼륨)

* **개념:** 파드가 실행되고 있는 **실제 물리 서버(노드)의 디스크 공간에 데이터를 저장**하는 볼륨입니다.
* **특징:** 파드가 삭제되더라도 노드의 디스크에 데이터가 물리적으로 남아있기 때문에 안전합니다. 주로 노드의 로그 시스템이나 도커 호스트 자체의 파일 시스템에 파드가 접근해야 할 때 사용합니다.

### 3. 🌐 PersistentVolume (영구 외부 볼륨)

* **개념:** 클러스터나 노드 외부의 **독립적인 외부 스토리지 서버에 데이터를 저장**하는 볼륨입니다.
* **특징:** 파드가 완전히 삭제되거나, 다른 노드로 이사를 가더라도 외부 저장소에 데이터가 안전하게 보존되므로, 데이터베이스(DB)처럼 영구적인 보존이 필요한 중요한 데이터를 다룰 때 필수적으로 사용됩니다.

---

### 💡 한 줄 요약

* **emptyDir:** 파드랑 같이 살고 같이 죽는 **임시 대피소**
* **hostPath:** 내가 살고 있는 서버 방 한구석에 있는 **붙박이장**
* **PersistentVolume:** 서버가 터져도 안전하게 보관되는 **외부 은행 금고**

### PV(PersistentVolume)
- 외부 스토리지를 의미
- 쿠버네티스 클러스터 관리자가 PV를 생성/관리하고, 사용자는 PVC를 통해 PV를 요청한다.
  - PV(PersistentVolume): 스토리지 자원
  - PVC(PersistentVolumeClaim): 스토리지를 동적으로 바인딩하기 위한 요청 객체
  - NFS(Network File System) 서버: 원격 서버에 존재하는 파일들을 내 컴퓨터에 있는 것처럼 사용하는 것
    - nfs-common 설치 필요
    - systemctl status nfs-server.service

## 🔍 그림 9-30의 핵심 컴포넌트 구조

### 1. 🟢 노드 내부 (myserver02)

그림의 왼쪽 박스는 실습 중 파드가 생성되어 달리고 있는 **`myserver02` 워커 노드**의 내부 상황을 보여줍니다.

* **`nginx-volume-04` (파드):** 실제 실행 중인 컨테이너입니다.
* **`/mount01`:** 컨테이너 내부에 만들어진 가상의 폴더 경로입니다. 이 안에 사용자가 `nfs_pvtest.txt`라는 파일을 생성했습니다.

### 2. 🟡 쿠버네티스 중개 구역 (PVC와 PV)

가운데 박스는 쿠버네티스가 파일 시스템을 어떻게 추상화하여 연결했는지 보여줍니다.

* **`pvc-01` (신청서) ➡️ `pv-01` (방):** 개발자가 작성한 `pvc-01` 신청서가 관리자가 만든 `pv-01` 볼륨과 끈끈하게 연결(`Bound`)되어 있습니다.
* 파드의 `/mount01` 폴더로 들어간 파일(`nfs_pvtest.txt`)은 이 두 관문을 차례대로 통과하게 됩니다.

### 3. 🟠 실제 물리 저장소 (NFS 서버: 10.0.2.6)

그림의 오른쪽 박스는 네트워크 너머에 있는 `myserver03` 노드(NFS 서버)를 가리킵니다.

* **`/tmp/k8s-pv`:** 가상 세계를 통과한 파일이 최종적으로 물리적으로 저장되는 실제 디스크 공간입니다.
* 파드 안에서 만들었던 `nfs_pvtest.txt` 파일이 화살표를 타고 국경을 넘어 **실제 NFS 서버의 디렉터리 안에 안전하게 저장**되어 있는 모습을 시각적으로 보여줍니다.

## 💡 이 그림의 진짜 의미와 하단 명령어

이 그림이 뜻하는 본질은 "파드가 다른 노드(`myserver02`)에서 실행 중이더라도, 데이터는 네트워크를 타고 완전히 독립된 외부 서버(`10.0.2.6`)에 저장된다"는 것입니다.

이 원리 덕분에 그림 하단에 나오는 명령어처럼 ❶~❸ 파드와 PVC, PV를 **전부 삭제(`kubectl delete`)하더라도**, 오른쪽 NFS 서버 디스크에 저장된 `nfs_pvtest.txt` 파일은 삭제되지 않고 영구히 안전하게 남게 됩니다. (PV에 설정했던 `persistentVolumeReclaimPolicy: Retain` 정책 덕분입니다.)
