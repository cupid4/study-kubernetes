# 9.6.2 헬름의 개념
 - yaml 파일을 만들지 않고, 쿠버네티스 환경에서 애플리케이션을 쉽고 편하게 설치할 수 있다.

![alt text](<asset/09.6.2 헬름의 개념/image.png>)


공유해주신 370페이지의 [그림 9-34 헬름의 개념]은 쿠버네티스의 패키지 매니저인 Helm(헬름)이 어떻게 작동하고 구성되는지 한눈에 보여주는 아키텍처 지도입니다.

쉽게 말해, 스마트폰에서 앱을 설치할 때 '앱스토어'에서 다운받아 클릭 한 번으로 설치하듯, 쿠버네티스에서도 복잡한 여러 개의 YAML 파일들을 하나로 묶어 쉽게 설치하고 관리할 수 있게 도와주는 **'쿠버네티스용 앱스토어'** 시스템입니다.

그림에 나오는 3가지 핵심 영역을 기준으로 역할을 설명해 드릴게요.

---

## 1. 🌐 헬름 차트 리포지토리 (Helm Chart Repository)

* **역할:** 전 세계 개발자들이 미리 만들어 놓은 쿠버네티스 실행 패키지(예: Nginx, MySQL, WordPress 등)들이 모여 있는 공용 저장소(앱스토어)입니다.
* **동작:** 도커 허브와 비슷하게, 내가 만든 패키지를 저장소에 올릴 수도 있고(`push`), 필요한 패키지를 내 환경으로 다운로드(`pull`) 받아올 수도 있습니다.

## 2. 🧑‍💻 관리자 & 헬름 클라이언트 (Helm Chart)

관리자(사용자님)가 내 PC나 터미널에서 명령어를 내리는 구역입니다. 이곳에서 다운로드한 '헬름 차트'를 열어보게 되는데, 내부 구조가 크게 두 가지로 나뉩니다.

* **템플릿 (Template):** 기본 구동에 필요한 복잡하고 고정된 쿠버네티스 YAML(Service, Deployment 등)들의 뼈대 소스코드입니다.
* **values.yaml (설정 파일):** **이 그림에서 가장 중요한 파일입니다.** 복잡한 템플릿 코드를 일일이 수정할 필요 없이, 이 파일 안에 있는 몇 가지 변수(예: '서버 개수는 3개', '포트 번호는 80')만 내 입맛대로 수정하면 나만의 맞춤형 패키지가 완성됩니다.

## 3. ☸️ 쿠버네티스 클러스터 (마스터 노드)

* **역할:** 최종적으로 내 애플리케이션이 실행되는 실제 서버 환경입니다.
* **동작:** 헬름 클라이언트에서 `헬름 설치(helm install)` 명령을 내리면, 커스텀된 헬름 차트가 쿠버네티스의 '마스터 노드'로 전달됩니다. 책의 본문 설명처럼 워커 노드마다 일일이 설치할 필요 없이, 대문 역할인 마스터 노드가 명령을 받아 전체 클러스터에 앱을 배포하게 됩니다.

---

## 💡 그림으로 보는 전체 흐름 요약

1. **Pull:** 리포지토리(앱스토어)에서 원하는 프로그램의 '헬름 차트'를 다운로드한다.
2. **Modify:** `values.yaml` 파일을 수정해 내 환경에 맞게 옵션(포트, 용량 등)을 커스텀한다.
3. **Install:** 마스터 노드에 설치 명령을 내려 쿠버네티스 클러스터에 한 번에 배포한다.
