# 9.2.5 스케일 (Scale)

쿠버네티스에서 스케일(Scale)은 시스템의 부하에 맞춰 **파드(Pod)의 개수를 늘리거나(Scale-out) 줄이는(Scale-in) 작업**을 의미합니다. 앞서 배운 디플로이먼트 YAML 파일의 `replicas` 수치를 수정하여 간편하게 조정할 수 있습니다.

## 1. 스케일 실습을 위한 디렉터리 준비

이전 실습 파일(`ex02`)을 그대로 복사하여 새로운 실습 디렉터리(`ex03`)를 구성합니다.

```bash
eevee@myserver01:~$ cd work/ch09/                      # ❶ 9장 실습 디렉터리로 이동
eevee@myserver01:~/work/ch09$ ls                        # ❷ 현재 디렉터리 확인 (ex01, ex02 존재)
ex01 ex02
eevee@myserver01:~/work/ch09$ cp -r ex02 ex03           # ❸ 기존 ex02 디렉터리를 통째로 복사해 ex03 생성
eevee@myserver01:~/work/ch09$ ls                        # ❹ 생성 확인 (ex03 추가 완료)
ex01 ex02 ex03
eevee@myserver01:~/work/ch09$ cd ex03/                  # ❺ ex03 디렉터리로 이동
eevee@myserver01:~/work/ch09/ex03$ ls                   # ❻ 복사된 deploy-test01.yml 파일 확인
deploy-test01.yml

```
